# 校园二手交易平台 — 项目结构说明

## 技术栈

| 层 | 技术 |
|------|------|
| 前端 | Vue 3 + Vite |
| 后端 | Spring Boot 3 + Java 22 |
| 数据库 | MySQL 8.0 + MyBatis-Plus |
| 部署 | 阿里云 ECS（47.96.152.231:8080） |

---

## 项目整体结构

```
campus-trade/
├── campus-trade-server/          ← 后端（Spring Boot + Java）
│   ├── src/main/java/com/campustrade/
│   │   ├── CampusTradeApplication.java  ← 项目启动入口
│   │   ├── config/                      ← 配置类
│   │   ├── controller/                  ← 接收请求、返回数据
│   │   ├── service/impl/                ← 业务逻辑（核心！）
│   │   ├── entity/                      ← 数据库表映射
│   │   ├── mapper/                      ← 数据库操作
│   │   ├── dto/                         ← 请求/返回数据格式
│   │   └── exception/                   ← 全局异常处理
│   ├── src/main/resources/
│   │   ├── application.yml              ← 配置文件
│   │   └── static/                      ← 前端打包产物
│   └── pom.xml
│
└── campus-trade-ui/                    ← 前端（Vue 3）
    └── src/
        ├── main.js                     ← 前端入口 + 路由
        ├── App.vue                     ← 底部导航栏
        ├── api/index.js                ← 所有后端接口调用
        ├── views/                      ← 页面组件
        ├── components/                 ← 复用组件
        └── assets/                     ← 静态资源
```

---

## 数据流向

```
浏览器 → API请求 → Controller → Service → Mapper → MySQL
                                                       ↓
浏览器 ← JSON返回 ← Controller ← Service ← Mapper ←──┘
```

通俗理解：
- **Controller** = 前台接待，收快递发快递
- **Service** = 后厨做饭，所有逻辑在这
- **Entity** = 食材清单（数据库表的映射）
- **Mapper** = 仓库管理员（查/插入数据）

## 后端模块详解

### entity/ — 数据库表映射（7张表）

| 文件 | 对应表 | 说明 |
|------|--------|------|
| User.java | t_user | 用户（用户名、密码、昵称、角色） |
| Product.java | t_product | 商品（标题、价格、图片、视频、分类） |
| Order.java | t_order | 订单（买家/卖家、金额、状态） |
| Review.java | t_review | 评价（评分1-5、内容、关联订单） |
| Favorite.java | t_favorite | 收藏（用户-商品关联） |
| Message.java | t_message | 私信（发信人、收信人、内容） |
| Announcement.java | t_announcement | 公告（管理员发布） |

### controller/ — 接收请求

| 文件 | 请求路径 | 功能 |
|------|----------|------|
| UserController.java | /api/user/** | 登录、注册、修改资料 |
| ProductController.java | /api/product/** | 发布、列表、详情、下架 |
| OrderController.java | /api/order/** | 下单、支付、发货、收货 |
| ReviewController.java | /api/review/** | 提交评价、查看评价、好评率 |
| FavoriteController.java | /api/favorite/** | 收藏/取消 |
| MessageController.java | /api/message/** | 发私信、查看消息 |
| FileController.java | /api/file/** | 图片/视频上传 |
| AiController.java | /api/ai/** | AI智能助手 |
| AnnouncementController.java | /api/announcement/** | 公告管理 |
| AdminController.java | /api/admin/** | 管理后台 |

### service/impl/ — 业务逻辑

| 文件 | 核心逻辑 |
|------|----------|
| UserServiceImpl | 注册校验、登录验证 |
| ProductServiceImpl | 商品检索分页、图片URL拼装 |
| OrderServiceImpl | **下单乐观锁**、支付状态流转 |
| ReviewServiceImpl | 评价去重、好评率计算 |
| FavoriteServiceImpl | 收藏切换 |
| MessageServiceImpl | 会话列表、未读数统计 |

### 重要技术点

| 技术 | 位置 | 说明 |
|------|------|------|
| 乐观锁 | OrderServiceImpl.create() | 防止并发重复下单 |
| 原子SQL | ProductServiceImpl.addViewCount() | 防止浏览量并发错误 |
| 全局异常 | GlobalExceptionHandler | 统一错误处理 |
| 参数校验 | DTO类中的@NotBlank/@Size | 接口参数验证 |
| API Key安全 | application.yml用\${DEEPSEEK_API_KEY} | 环境变量注入，不硬编码 |
