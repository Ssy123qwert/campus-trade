# 校园二手交易平台 - 项目总结

## 项目概况

校园二手交易平台是一个基于 Spring Boot + Vue 3 的 Web 应用，支持商品发布、浏览、搜索、交易、收藏、即时通讯等功能，已成功部署到阿里云服务器并正常运行。

## 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot | 3.2.5 |
| ORM | MyBatis-Plus | 3.5.6 |
| 数据库 | MySQL | 8.0 |
| Java | OpenJDK (Zulu) | 22.0.2 |
| 前端框架 | Vue 3 + Vite | - |
| 构建工具 | Maven | - |
| 工具库 | Hutool, FastJSON2, Lombok | - |
| AI 对话 | DeepSeek API | v1 |

## 功能模块

- 用户模块：注册、登录、个人信息管理
- 商品模块：发布、浏览、搜索、分类筛选、排序、下架
- 订单模块：创建订单、支付、状态流转（待支付→已支付→已发货→已收货）
- 收藏模块：收藏/取消收藏、查看收藏列表
- 消息模块：买卖双方即时通讯、未读消息提醒
- 公告模块：平台公告发布与管理
- 文件上传：图片上传与访问
- 管理后台：用户管理、商品管理、公告管理
- AI 对话：集成 DeepSeek 聊天 API

## 部署过程中遇到的问题及解决

### 1. 端口安全组未放行
- 问题：外部无法访问 8080 端口
- 解决：在阿里云控制台安全组中添加入方向规则，放行 TCP 8080 端口

### 2. JAR 包不完整
- 问题：服务器上 jar 只有 277KB，缺少 Spring Boot 启动入口，报 `no main manifest attribute`
- 原因：本地 Java 进程锁定了 target 目录下的 jar 文件，导致 Maven 无法完整打包
- 解决：杀掉占用进程（PID 6168, 21056）后重新编译，生成 33MB 完整 jar 包

### 3. API 路径不匹配
- 问题：访问 `/api/products` 报 500 错误，日志显示 `No static resource api/products`
- 原因：后端 Controller 映射的是 `/api/product`（单数），测试时用了 `/api/products`（复数）
- 解决：使用正确路径 `/api/product/list` 等，API 正常工作

### 4. 图片无法显示
- 问题：前端页面能显示商品数据但图片空白
- 原因：服务器 `/opt/campus-trade/uploads/images/` 目录为空，图片文件未同步；配置文件缺少 `campustrade.upload.path` 配置项
- 解决：将本地 6 张图片通过 SFTP 上传到服务器，补充配置文件中的上传路径配置

### 5. 配置文件项名不一致
- 问题：FileController 使用 `@Value("${campustrade.upload.path}")`，但服务器 application.yml 中只有 `upload.path`
- 解决：在 application.yml 中添加 `campustrade.upload.path: /opt/campus-trade/uploads`

## 后续维护

如需修改代码并更新服务器：
1. 在本地修改代码
2. 运行 `mvn clean package -DskipTests` 编译新 jar
3. 通过 SFTP 上传 jar 到服务器 `/opt/campus-trade/`
4. SSH 执行 `pkill -f campus-trade-server && cd /opt/campus-trade && nohup java -jar campus-trade-server-1.0.0.jar --spring.config.location=file:/opt/campus-trade/application.yml > app.log 2>&1 &`
